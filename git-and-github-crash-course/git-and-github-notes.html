Git & Github Crash Course For Beginners

Commands

  • git init Initializes a local git repository.
  • git add <file> Adds the specified file or files to the index, use a "." to denote all files in the pwd.
  • git status Checks the status of the working tree.
  • git commit Commit your changes in the index to the remote repository. Adding "-m" will allow you to provide a description of the changes you are committing.
  • git push Push to your remote repository. Adding -u origin <branch-name> will push your indexed files in your local repository to your remote repository.
  • git pull Pulls the latest changes from your remote repository
  • git clone Clones a repository into a new directory.
  • git config --global user.name <username> Links your github username for use with git commands.
  • git config --global user.email <email> Links your email address that you used for github for use with bash commands.
  • git rm --cached <file> Removes a file from the staging area if you don't want them to be indexed.
  • git branch <branch-name> Create/change to a different branch in the repository.
  • git checkout <branch-name> Allows you to switch between the different branches in the repository.
  • git merge <branch-name> Merges the changes from a branch to the main branch.
  • git remote add origin <remote-repository-url> Links a remote repository to the local repository on your machine.

Notes:

  • Git is able to keep track of your code history by taking snapshot of your files and you can decide if these snapshots will be applied to the main remote repository.
  • Git is available on windows, linux, and macos. Macos and Windows use a basic installer while linux uses package commands to download them.
  • It is recommended to use the Bash shell as opposed to the GUI to better understand how Git works.
  • For windows, you can also use the CMD or powershell to run all of the git commands but it is best to use the bash shell instead.
  • You can create a file labeled ".gitignore" to prevent git add from indexing any file listed in .gitignore. Simply adding the file name or an entire directory name in the .gitignore file will prevent it from being indexed even when all files are specified with the git add command
  • The main branch in the repository is now labeled as main and no longer listed as master, so be sure to specify the branch name as main when committing to the remote repository.
  • Branches can be used to work parallel to the main branch to prevent anyone person from over writing work from another person.
  • It is always useful to have a readme.md file in your repository to provide insight for viewers by displaying a description of your project.
  • Remote repositories are separated via the initialized git folder that was created when using git init. You can name the remote repository something other than origin which is useful if you have multiple remote repositories to add. If you have separated projects though it is unnecessary to do so.

Examples:

Initializing a Directory

Before doing anything, you first have to add your username and email address for the bash shell using the git config --global commands. Afterwards, the first command you will use will be git init,this will create an empty git repository in your folder to allow you to use git commands. Then you would add the files you want to index with the git add <file> command, usually you will use "git add ." to add all files in your current pwd. From here you would then commit the changes with the git commit command.
