RegEx Notes:

  • To utilize regex, you need to create a literal expression which is done by entering a pattern you want to search for between two forward slashes
    Example, /test/ will look for the characters "test" within a string. The values within the //, in this case 'test', are called tokens.
  • By default, an expression will only search for the first occurrence of a pattern within a string, but this can be changed by adding flags
    There are various flags that can be added after the last /, like g for global which will show all occurrences of the specific pattern.
    Another useful flag that can be added at the end of a regex pattern is i, which is case sensitivity so it will ignore case within the pattern.
  • More complex search patterns can also be created by using special characters, like ".", "^", "$", and more, which allows more dynamic patterns.
    Another useful special character is "+" and when used within the regex pattern it signifies that the pattern requires one or more of the proceeding variable.
    The "?" special character signifies that the pattern is optional, meaning that the proceeding token has one occurrence or none, basically it is like an optional flag.
    The "*" special character is a combination of the "+" and "?" since it signifies that the proceeding token is optional or can occur more than once.
    The "." special character functions as a wild card, it will match any value in its place in the string aside from a new line.
    The "^" special character is used to specify to look for a token at only the start of the string of text and the "$" does the opposite in which it looks for a token only at the end of the string.
  • Within a regex pattern, a "\" can be used to escape any special characters to allow them to be used within a search pattern.
  • Another important thing are character classes, these allow the regex pattern to match to any specific set of characters.
    Example, "\w" signifies the word class so any alphabetical, number, or "_" character would be flagged by the regex pattern.
    Other character classes are "\s" which matches any form of whitespace and "\d" which matches any digit.
    Also, all of these have opposing versions, so simply capitalizing character will have it match the opposite of its character class, like a "\S" would match everything expect whitespace.
  • "{}" can be used to specify how many occurrences of a token within the pattern should be looked for and adding a comma within the braces allows you to do a min and max value, or only a min if you don't add a value after the comma.
  • "[]" can be used to allow any value within the braces to be match in the specific spot. This allows you to specify a range like [a-z] to allow only lowercase characters to be at said specific point in the pattern.
  • "()" can be used to isolate tokens to a specific part of the pattern, which is useful when using things like '|' or ranges, like if you wanted to make sure the first character was a 't' or a 'T'. You could type /(t|T)he/. This will allow either an uppercase or lowercase "t" to still flag the pattern.
  • There are also options called lookAround, which allows you to find all proceeding or following characters after a specific pattern is found.
    This can be useful if you are trying to grab certain areas of the text the occur after certain patterns, without requiring you to flag or interact with said pattern, only the characters after or before it.
  • Groupings can be given a name by using '?<>' within the braces, '()'.
  • When using .replace() the first parameter is the pattern being tested and the second parameter is what you will replace the pattern with.
    Special characters like a "$" can be used to reference the original text or group, '$&' will reference the entire string or '$1' will reference only the first group within the pattern.
