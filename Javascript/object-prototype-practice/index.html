Object Prototype Notes:

  • A prototype is used to provide access to certain methods that are already mapped or available to other objects. The ability to do this is useful in the sense that once an object already has a specific method, it can be mapped to a similar object rather than creating repeat code for the other object.
  • Every object has a prototype property, whether it be a function, array, or an object. Even user created objects have their own prototype properties.
  • To access this property you enter the item's name, like User for a object, and append '.prototype'
    Note that only the constructor of an object can access the prototype property, not any of the objects created from it.
    Instead, they have the __proto__ property.
  • To attach methods to the prototype property, you enter the item and with the appended '.prototype' and then append a period followed by the name of the method you will be adding.
    Example would look like User.prototype.login = function(). This would added to the User object constructor prototype a method called login that would run a function.
  • Inheritance is useful with prototypes since it allows the creation of other objects that are similar while basing them off an already made object.
    Like with a User object being the base object for an Admin object that would also have a username and email, but additional properties and methods as well.
  • When utilizing Inheritance, rather than having to rewrite the variables for the inherited objects in the parameter, you can instead insert '...args' in the new object's parameter
    This will accept all parameters passed in and insert them into an array
  • In the constructor of the new object, you would use the '.apply' method on object being inherited.
    For this to work, you will need to pass in a parameter of what the 'this' keyword will be referring to within the inherited object, more often than not, it will be the 'this' since it will be used to reference the current object constructor.
    Example, an admin object would use the User.apply(this, args) within its constructor to inherit the properties and methods from the User object constructor. This will allow the User Object Constructor to reference the Admin Object and the necessary parameters to initialize the new object.
  • The prototype chain is exactly as it states, it is a chain of prototypes that an element has available to it.
    For example, the Admin Object would have a prototype chain of Admin.prototype to User.prototype to Object.prototype, note that Object.prototype is created by javascript itself.
    Having this chain allows the Admin Object to utilize ant of the methods or properties created by any of these prototypes.
