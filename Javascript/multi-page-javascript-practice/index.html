JavaScript Multi-Page Files (Import & Export Features) Notes

  • The import and export feature allows multiple javascript files to be utilized within a single webpage.
    The only difference in assigning these script files to the webpage is to provide them with type="module" to allow them to utilize the import and export features.
  • Export is used make functions, variables, or entire class objects available in other files.
    Import is used to make these function usable within other files via the import feature.
  • When exporting files, the template for export is to write "export { 'item name to be exported' }" and multiple items can be exported by adding commas after each item.
    There is no wildcard when it comes to exporting, so the best way to export everything in a module is to start add the word export before any variable or function that you know you will end up wanting exported.
    From here, you can luckily use the wildcard to import everything from a module at once.
  • When importing from files, you can either use deconstruction to specify a single variable or function to import, and you can use the "as ..." keyword to provide an alias to the import.
    You can also utilize the wildcard "*" character to import everything from another module. Again, you can use the "as ..." keyword to provide an alias for the entire module.
    Example, writing "import * as template from "./template.js" would provide access to all exported items from the template file and to access it all you would need to do is type "template.'item or function'".
