Local Storage Practice & Notes

Various Commands

  • setItem - Stores an item in LocalStorage
    For this to work, the item must be stored with a key, such as user or score, and the item itself must be a string. Things like objects and arrays must be stringified before being saved to LocalStorage.
  • JSON.stringify - Takes in an parameter and converts it to a string, this is necessary if an object, array, or even a number is being stored in local storage.
  • JSON.parse - Takes a string and converts it back into it original format, such as an object or an array.
    This is necessary if pulling a stringified item from LocalStorage, note that this also includes any number since the only thing localStorage can save is text or a string.
  • getItem - Retrieves an item from localStorage based on the key value entered into its parameter.
    This is used to retrieve an item from localStorage, the value entered into its parameter is what determines what information it pulls from localStorage.
  • removeItem - Removes an item from localStorage based on the key entered into its parameter.
    This is used to remove a specific item from localStorage, note that the key value itself is also removed from localStorage, as opposed to it just setting it equal to an empty string.
  • clear() - This is a method used to remove everything in localStorage. This is useful when you want to remove everything without have to call each individual key pair with the removeItem method, but it can be problematic if you wish to keep some of the info since it doesn't have the ability to be selective when removing items.
  • StorageEvent - This is an event that is triggered after adding an event handler to the window element, or the main page, using the "storage" event. This seems to only be triggered if the user edits localStorage values from within the console or actual localStorage file, using the setItem or removeItem methods don't seem to trigger it.
